# MSC500.zip — Microsoft C 5.0 oracle distribution

This file documents the contents of `MSC500.zip`, the
[Microsoft C 5.0][release] DOS toolchain we use as the oracle when
building out `crates/msc` (the second reimplementation target — see
[`specs/plans/SECOND_COMPILER.md`](specs/plans/SECOND_COMPILER.md)).

[release]: https://winworldpc.com/product/microsoft-c/50

**`MSC500.zip` itself is gitignored** because Microsoft has never
licensed the binaries for redistribution. The companion
[`MSC500.sha256`](MSC500.sha256) is the integrity anchor: if a
contributor's MSC 5.0 install hashes to the same values, they have
the same toolchain we used to capture goldens, and their `crates/msc`
runs will produce byte-identical OBJs.

## How to acquire MSC 5.0

Microsoft C 5.0 (1987-10-15) shipped on five 5.25" or 3.5" floppy
disks. It's been treated as abandonware for ~30 years and is widely
available; we have no opinion on jurisdiction-specific legal exposure.

Practical sources:

- [WinWorldPC](https://winworldpc.com/product/microsoft-c/50) —
  curated, stable, hosts both floppy images and an extractable
  archive. The most popular source.
- [archive.org](https://archive.org) — search "Microsoft C 5.0";
  multiple uploads with varying provenance.
- [PCjs Project](https://www.pcjs.org/software/pcx86/lang/microsoft/c/5.00/)
  — Jeff Parsons' DOS software collection.

## Extraction layout

The zip's flat top level mirrors the layout BC2.zip uses, so the
`oracle` crate can drive both with the same DOSBox-mount approach:

```
BIN/        compiler driver (CL.EXE) + passes (C1, C1L, C2, C3) +
            MASM-era tooling (LINK.EXE, LIB.EXE, MAKE.EXE, CV.EXE).
            Plus QuickC (QC, QCL, QLIB) — unused by the oracle but
            shipped in the same distribution.
BIN/PATCH/  Stack-overflow patcher scripts (legacy DOS-era utility).
BIN/SAMPLE/ Demo programs (CFLOW.C, LIFE.C, etc.) — not needed by
            the oracle but kept for completeness.
INCLUDE/    Standard headers (stdio.h, string.h, dos.h, ...).
INCLUDE/SYS/ POSIX-style sub-headers (sys/types.h, sys/stat.h, ...).
LIB/        Runtime libraries. **Small model only** in this
            distribution — see below.
MIXLANG/    Mixed-language (C ↔ FORTRAN/Pascal) header stubs.
TMP/        Empty.
```

### Memory-model coverage

`LIB/` contains only the small-model runtime variants:

- `SLIBC.LIB`  — small, no math emulation
- `SLIBCE.LIB` — small, math emulator (the typical default)
- `SLIBC7.LIB` — small, requires 8087
- `SLIBCA.LIB` — small, 8087 alternate
- `SLIBFA.LIB`, `SLIBFP.LIB` — FORTRAN-callable variants
- Plus `87.LIB`, `EM.LIB`, `LIBH.LIB`, and standalone helper OBJs

The medium/compact/large/huge runtime libraries (`MLIBCE.LIB`,
`CLIBCE.LIB`, `LLIBCE.LIB`, `HLIBCE.LIB`) are **not** in this zip.
That's enough for hello-world bring-up and the small-model bring-up
corpus, but matches against the BCC corpus's large-model fixtures
will require finding a more complete MSC 5.0 distribution or
recompiling the runtime from MS's source-disk release.

## Verifying a candidate install

```bash
# 1. Extract your MSC500.zip somewhere.
unzip -d /tmp/msc500 MSC500.zip

# 2. Hash-check against the manifest.
cd /tmp/msc500
sha256sum -c <path-to-repo>/MSC500.sha256
```

Every entry should report `OK`. A single mismatch means your copy
differs from ours and goldens captured against it will diverge
trivially even if the compiler behavior is functionally identical.

## Notes

- File timestamps inside the zip are `1987-10-15 05:00` — Microsoft
  C 5.0's official release date. The `2026-05-16` directory entries
  are filesystem metadata from the local extract that produced the
  zip; they don't affect the SHAs of the contained files.
- The oracle's `microsoft-msc5` profile (TBD — see the second-compiler
  plan) will lazy-extract `MSC500.zip` into `.msc500/` (gitignored,
  mirrors the `.bc2/` pattern) and drive `CL.EXE` inside DOSBox with
  `INCLUDE=C:\MSC500\INCLUDE; LIB=C:\MSC500\LIB`.
